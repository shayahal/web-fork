# Add Blog Post Instructions

## Purpose
Automate the process of adding a blog post from Obsidian Vault to the Gatsby site in both Hebrew and English versions.

## Source and Destination
- **Source folder**: `/Users/shayyahal/Documents/Obsidian Vault/Writing/Prod`
- **Destination folder**: `/Users/shayyahal/Code/web-fork/content/blog`

## Process Steps

### 1. Read the source file
- The markdown file should be in the Obsidian Vault folder
- File format: Plain markdown without frontmatter

### 2. Generate slug from filename
- Convert Hebrew filename to URL-friendly English slug
- Example: "אירועים לכל עת - לוגיסטיקה" → "event-organization-logistics"
- Use lowercase, replace spaces with hyphens, remove special characters

### 3. Create folder structure
```bash
mkdir -p "content/blog/{slug}"
mkdir -p "content/blog/{slug}-en"
```

### 4. Create Hebrew version (`content/blog/{slug}/index.md`)
```markdown
---
title: "{title in Hebrew}"
date: '{YYYY-MM-DD}'
description: '{description in Hebrew}'
direction: rtl
language: 'he'
tags: [{tag1}, {tag2}, {tag3}]
---

{content from source file}
```

### 5. Create English version (`content/blog/{slug}-en/index.md`)
- Translate the content to English
- Use the same structure as Hebrew version but with:
  - `direction: ltr`
  - `language: 'en'`
  - English title and description

### 6. Required frontmatter fields
- **title**: Blog post title (string)
- **date**: Publication date in 'YYYY-MM-DD' format
- **description**: Short description (1-2 sentences)
- **direction**: 'rtl' for Hebrew, 'ltr' for English
- **language**: 'he' for Hebrew, 'en' for English
- **tags**: Array of tags (e.g., ['hosting', 'tips', 'thoughts'])

### 7. Naming conventions
- Hebrew version folder: `{slug}` (e.g., `event-organization-logistics`)
- English version folder: `{slug}-en` (e.g., `event-organization-logistics-en`)
- Both use `index.md` as the filename

### 8. Content formatting
- Preserve markdown formatting from source
- Ensure quotes use proper format (״״ for Hebrew, "" for English)
- Maintain paragraph breaks and list structure

### 9. Translation guidelines
- Translate naturally, not word-for-word
- Adapt cultural references when necessary
- Keep technical terms consistent (e.g., "finger food", brand names)

## Example Command Structure

```bash
# Usage: add-blogpost <filename-in-obsidian> <slug> <tag1,tag2,tag3>
# Example: add-blogpost "אירועים לכל עת - לוגיסטיקה.md" "event-organization-logistics" "hosting,tips,thoughts"
```

## Post-creation steps
1. Restart Gatsby development server to pick up new content
2. Verify both versions appear in blog lists
3. Check RTL/LTR rendering is correct
4. Commit with message: "Add blog post: {title}"

## Notes
- Date should be set to current date or specified date
- Tags should be provided as comma-separated values
- Always create both Hebrew and English versions
- Hebrew version comes from source file as-is
- English version needs to be translated (can prompt Claude for translation)
